   - `limit`: number, optional, default 20, min 1, max 100. Use `@IsOptional() @Type(() => Number) @IsInt() @Min(1) @Max(100) @Transform(({ value }) => { const n = parseInt(value); return isNaN(n) || n < 1 ? 20 : Math.min(n, 100); })`.

**attendance.service.ts — update `listRecords` method:**

1. Add `pagination: PaginationDto` as the last parameter (after `managerId?: string`).
2. Import `PaginationDto` and `PaginatedResult` from `'../common/dto/pagination.dto'`.
3. Change return type from `Promise<Record<string, unknown>[]>` to `Promise<PaginatedResult<Record<string, unknown>>>`.
4. After building the query chain (company_id filter, date range, userId/employeeIds filters, order), do NOT call `.range()` yet. Instead, add `{ count: 'exact' }` to the `.select()` call:
   ```typescript
   .select(`*, users!user_id ( full_name, email )`, { count: 'exact' })
   ```
5. Then chain `.range((pagination.page - 1) * pagination.limit, pagination.page * pagination.limit - 1)`.
6. Destructure `{ data, error, count }` from the query result.
7. Return `{ data: (data ?? []) as Record<string, unknown>[], total: count ?? 0, page: pagination.page, limit: pagination.limit }`.

Note: The two early-return paths (when managerId is set and no divisions/employees exist) must also return the paginated shape:
