**attendance.service.ts — update `getMonthlyReport` method:**

The tricky constraint: stats (total, lateCount, etc.) MUST reflect the full month dataset, not just the current page. The current implementation computes stats from `reportRecords` (all records in memory). With pagination, fetch ALL records for stats but return only the page slice for `records`.

Implementation approach — two-step query:
1. Fetch the full dataset for stats computation (no range, just fetch all with the same filters).
2. Apply `.range()` for the records slice returned to the client.

Actually, a more efficient approach: fetch the count separately and apply `.range()` for records, but compute stats from records only if doing a full sweep. Because stats need every record's `check_in_status` and `missing_checkout` — a count query won't provide that.

Use this approach:
1. Run the FULL query (no range) to get all records for stats. Add `{ count: 'exact' }` to `.select()` to get total in one round-trip.
2. Slice the `reportRecords` array in memory for the page: `reportRecords.slice((page-1)*limit, page*limit)`.
3. Compute stats from the full `reportRecords` array (unchanged from current logic).
4. Return `{ records: pagedRecords, stats, total: reportRecords.length, page: pagination.page, limit: pagination.limit }`.

This keeps stats correct and avoids a second DB query. The tradeoff (fetching all records for stats) is acceptable for now — monthly reports are bounded by month and manager scope.

Steps:
1. Import `PaginationDto` from `'../common/dto/pagination.dto'` (already imported in Task 2 of Plan 10-01 for listRecords — it's in the same file, so just verify the import is present).
2. Add `pagination: PaginationDto` as the last parameter to `getMonthlyReport`.
3. After `const reportRecords = data ?? [];`, compute `pagedRecords`:
   ```typescript
   const pagedRecords = reportRecords.slice(
     (pagination.page - 1) * pagination.limit,
     pagination.page * pagination.limit,
   );
   ```
4. Update the return value to add `records: pagedRecords` (instead of `records: reportRecords`) and add `total`, `page`, `limit`:
   ```typescript
   return {
     records: pagedRecords,
     stats: { total, lateCount, onTimeCount, withinGraceCount, missingCheckoutCount, lateRate },
     total: reportRecords.length,
     page: pagination.page,
     limit: pagination.limit,
   };
   ```
5. Update the two early-return paths (empty manager cases) to also include `total: 0, page: pagination.page, limit: pagination.limit`.
