<objective>
Migrate manager scoping in the attendance backend from `users.manager_id` to division-based membership, and add division assignment to the users update endpoint.

Purpose: In v2.0, a Manager's visibility is determined by which Divisions they manage, not by a direct `manager_id` on employee records. Every attendance query that was previously scoped by `users WHERE manager_id = managerId` must be updated to: (1) find the Manager's Divisions, (2) find employees in those Divisions, (3) scope attendance records to those employees.

This plan also adds `divisionId` to `UpdateUserDto` so Admins can assign employees to Divisions via the existing PATCH /users/:id endpoint.

Output: Updated `attendance.service.ts` (3 methods), updated `users.service.ts` + `update-user.dto.ts` to accept `divisionId`.
</objective>
