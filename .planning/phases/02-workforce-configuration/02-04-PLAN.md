DB table employee_shifts (created in 003_workforce_config.sql):
```sql
employee_shifts(
  id UUID PRIMARY KEY,
  company_id UUID NOT NULL REFERENCES companies(id),
  user_id UUID NOT NULL REFERENCES users(id),
  shift_id UUID NOT NULL REFERENCES shifts(id),
  effective_date DATE NOT NULL,
  created_at TIMESTAMPTZ,
  UNIQUE (user_id, effective_date)   -- one assignment per user per date
)
-- RLS enabled: tenant isolation on company_id
```

Active shift logic: The shift with the LATEST effective_date that is <= today's date is the "active" shift.
Query pattern: SELECT * FROM employee_shifts WHERE user_id=$1 AND company_id=$2 AND effective_date <= CURRENT_DATE ORDER BY effective_date DESC LIMIT 1

    Create backend/src/shifts/shift-assignments.service.ts injecting SupabaseService:
    - assignShift(companyId: string, dto: AssignShiftDto):
      1. Verify shift exists and belongs to companyId: SELECT from shifts WHERE id=dto.shiftId AND company_id=companyId. If not found throw NotFoundException('Shift not found').
      2. Verify user exists and belongs to companyId: SELECT from users WHERE id=dto.userId AND company_id=companyId. If not found throw NotFoundException('User not found').
      3. Insert into employee_shifts: { company_id: companyId, user_id: dto.userId, shift_id: dto.shiftId, effective_date: dto.effectiveDate }
      4. On conflict error (UNIQUE violation on user_id + effective_date): throw ConflictException('An assignment already exists for this employee on this effective date. Choose a different date.')
      5. Returns created assignment record
    - getActiveShift(companyId: string, userId: string):
      1. SELECT employee_shifts.*, shifts.name, shifts.start_time, shifts.end_time, shifts.grace_period_minutes FROM employee_shifts JOIN shifts ON employee_shifts.shift_id = shifts.id WHERE employee_shifts.user_id=userId AND employee_shifts.company_id=companyId AND effective_date <= CURRENT_DATE ORDER BY effective_date DESC LIMIT 1
      2. Returns the row or null (no active shift is not an error — employee may not have one yet)
    - listAssignments(companyId: string, userId: string):
      1. SELECT * from employee_shifts JOIN shifts ON employee_shifts.shift_id=shifts.id WHERE employee_shifts.user_id=userId AND employee_shifts.company_id=companyId ORDER BY effective_date DESC
      2. Returns full assignment history

    Create backend/src/shifts/shift-assignments.controller.ts:
    - @Controller('shifts') @UseGuards(JwtAuthGuard)
    - POST /shifts/assign → assignShift(req.user.companyId, dto)
    - GET /shifts/assignments/:userId → getActiveShift and listAssignments(req.user.companyId, params.userId) — returns { activeShift, history }
